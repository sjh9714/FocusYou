# Data Recovery Tools

Focus You의 데이터 도구는 원격 전송 없이 로컬에서만 동작합니다. 모든 복구 작업은 사용자가 명시적으로 실행할 때만 시작됩니다.

작업이 끝나면 설정의 진단 섹션 또는 안전 모드 창에서 최근 작업 결과를 확인할 수 있습니다. 백업이나 진단 번들처럼 폴더가 생성되는 작업은 경로를 복사하거나 Finder에서 바로 열 수 있습니다.

## 백업 만들기

`설정 > 진단 > 데이터 도구 > 백업 만들기`는 Application Support 안의 Focus You 데이터 저장소와 진단 manifest를 선택한 폴더 아래 `FocusYouBackup-yyyyMMdd-HHmmss` 형식으로 복사합니다.

- 원본 store 파일은 삭제, 이동, 수정하지 않습니다.
- `diagnostics.json`에는 백업 시점의 파일 상태와 크기, 수정 시간이 포함됩니다.
- 백업 폴더는 그대로 보관하고, 복구 전에 가능하면 Finder에서 별도 위치에 한 번 더 복사해 두세요.

## 백업 미리보기

`백업 미리보기`는 선택한 백업을 임시 복사본으로 열어 데이터 개수를 확인합니다.

- 원본 백업 store를 직접 열지 않습니다.
- 프로필, 차단 사이트, 차단 앱, 세션, 스케줄, 배지 개수와 세션 날짜 범위를 표시합니다.
- corrupt/incompatible store는 현재 Application Support 경로를 건드리지 않고 실패 메시지만 표시합니다.

## 백업 가져오기

`백업 가져오기`는 정상 persistent store가 열린 상태의 설정 화면에서만 사용할 수 있습니다.

- 기본으로 가져오는 항목: 프로필, 차단 사이트, 차단 앱, 스케줄
- 선택하면 가져오는 항목: 세션 기록, 배지
- 기존 데이터는 덮어쓰거나 삭제하지 않고 새 프로필로만 추가합니다.
- 가져오기 전 미리보기에서 현재 데이터와 중복되어 건너뛸 세션/배지 수와 실제 새로 추가될 수를 확인할 수 있습니다.
- 이름이 충돌하면 `Name (Imported)`, `Name (Imported 2)` 순서로 새 이름을 붙입니다.
- 세션 기록은 같은 시작/종료 시각, 모드, 타입, 시간, 완료 상태를 가진 항목이 이미 있으면 중복으로 보고 건너뜁니다.
- 세션의 의도/회고 필드는 보존하지만, 백업의 오래된 Apple Calendar 이벤트 ID는 복사하지 않습니다.
- 배지는 `milestoneID` 기준으로 현재 없는 배지만 가져오고, 백업 안에 같은 배지가 여러 개 있으면 가장 이른 획득 시점 하나만 사용합니다.
- 가져오기 중 저장에 실패하면 현재 context를 rollback해 부분 import를 남기지 않습니다.

## 안전 모드에서 할 일

앱이 SwiftData 저장소 문제로 안전 모드로 시작하면 실제 가져오기는 제공하지 않습니다. 이 상태에서는 먼저 다음 작업만 수행하세요.

1. 데이터 백업 만들기
2. 진단 로그 내보내기
3. Application Support 폴더 확인
4. 앱 재시작

안전 모드의 in-memory store에는 변경사항이 영구 저장되지 않을 수 있으므로, 백업 가져오기는 앱이 정상 저장소로 다시 시작된 뒤 실행하세요.

## 진단 로그 내보내기

`진단 로그 내보내기`는 사용자가 직접 공유할 수 있는 로컬 진단 번들을 만듭니다.

- 차단 도메인, 앱 이름, 프로필명, 홈 디렉터리 경로처럼 민감할 수 있는 값은 redaction됩니다.
- 로그 수집에 실패해도 번들 생성은 가능한 범위에서 계속되고, 실패 사유가 manifest에 기록됩니다.
